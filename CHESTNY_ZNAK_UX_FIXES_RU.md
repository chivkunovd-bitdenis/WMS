# Честный знак — задачи на доработку (UX/комплаенс)

Источник: ревью всего модуля ЧЗ (фронт). Задачи нарезаны атомарно для Cursor.
Все пути — относительно корня репозитория. Номера строк указаны на момент ревью и могут сместиться — ориентируйся по именам функций/символов и `data-testid`.

**Целевая модель печати ЧЗ (зафиксирована с заказчиком):**
Печать только **по товару** через конструктор: оператор в задании на упаковку жмёт «Печать ЧЗ» на строке товара → открывается конструктор (`MarkingPrintDialog`) → печатается лента по этому товару.
**Конструктор — ключевая задуманная фича, его НЕ упрощать и НЕ трогать логику свободы.** Пользователь сам произвольно расставляет блоки (ШК маркета / ЧЗ) в любом порядке и с любым количеством копий. Пример: `ШК маркета → ЧЗ → ШК маркета → дубль того же ЧЗ` (одни клеят на пакет, другие на товар; кто-то печатает сначала оба ЧЗ, потом этикетки). Логика — дать юзеру выбрать.
**Раскладка должна сохраняться на учётке пользователя:** Вася выбрал свою раскладку — в следующий раз у него она же; у Пети — своя (см. T-A5).

**Два типа ШК и условное поведение окна печати (см. T-A7):**
- Есть два разных штрихкода: **ШК ВБ** (штрихкод маркетплейса) и **ШК ЧЗ** (датаматрикс Честного знака).
- Товар **с признаком ЧЗ** → конструктор с **обоими** блоками (ШК ВБ + ЧЗ), у каждого выбирается количество, и блоки переставляются на интерактивной ленте.
- Товар **без признака ЧЗ** → **только** простое окно с количеством ШК ВБ, **без конструктора** (этикетка одна — переставлять нечего).
- Количество — это **множитель на единицу**: итоговая лента = (кол-во товаров в упаковке) × (выбранное количество блока). Для ЧЗ это уже так; для ШК ВБ — сейчас абсолютное число, надо сделать множителем.

**Скан-печати, «Печати всех ЧЗ» и «Сверки пары ЧЗ» быть не должно** — лишние/ненужные механизмы, противоречащие процессу.

Порядок выполнения: сначала **EPIC 0** (фундамент), затем **EPIC A** (печать), затем **EPIC B** (комплаенс), далее остальное.

### T-A7 — Окно печати: разное поведение по признаку ЧЗ + два типа ШК (ШК ВБ / ЧЗ)

- **(B) Товар БЕЗ ЧЗ** → простое окно количества ШК ВБ (`ProductBarcodePrintDialog`), **без конструктора**.
- **(C) Количество = множитель на единицу:** лента = (кол-во товаров в упаковке) × (выбранное количество блока). Для ЧЗ уже так (`expandLayoutTape`: codes × copies). Для ШК ВБ без ЧЗ сейчас абсолютное число — сделать множителем (база — `qty_need_pack`/кол-во в упаковке).
**Файлы:** `frontend/src/components/MarkingPrintDialog.tsx`, `frontend/src/utils/markingPrintPresets.ts`, `frontend/src/components/ProductBarcodePrintDialog.tsx`, `frontend/src/components/FfProductLineCells.tsx`
**Что сделать:**
1. В конструкторе переименовать блок `label` → **«ШК ВБ»** (лейблы в билдере и пресетах; `data-testid` не менять). Подтвердить, что `label` рендерит именно ШК ВБ.
2. Убедиться, что для ЧЗ-строки в конструкторе доступны оба блока и количество (copies) каждого — уже есть, проверить.
3. Для не-ЧЗ строки печать остаётся простым окном количества (без конструктора) — поведение близко к текущему `ProductBarcodePrintDialog`.
4. Привести количество ШК ВБ к множителю «× кол-во в упаковке» в контексте упаковки.
**Свериться с бэком/заказчиком:**
- блок `label` = ШК ВБ (да/нет; если «этикетка товара» ≠ «ШК ВБ» — уточнить и завести отдельный тип блока).
- множитель «× в упаковке» — применять только в упаковке или и в каталоге (вне упаковки, вероятно, остаётся абсолютное число).
- «интерактивная лента» — сейчас порядок блоков меняется стрелками вверх/вниз + предпросмотр; если нужен именно drag-and-drop — завести отдельной задачей.
**Acceptance:**
- У ЧЗ-товара окно печати = конструктор с ШК ВБ и ЧЗ, количество каждого, перестановка порядка.
- У не-ЧЗ товара окно печати = только количество ШК ВБ, конструктора нет.
