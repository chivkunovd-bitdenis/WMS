# Эпики backlog (для нарезки GitHub Issues)

Используйте как основу для issues с шаблоном Feature: user story, AC (Given/When/Then), E2E сценарии. Метка `ready` — только когда AC и E2E описаны.

## E0 — Фундамент

- Мультитенантность: организация фулфилмента, селлеры, пользователи, роли.
- Auth: регистрация/логин email+пароль, сессии/JWT (решение на проектировании), изоляция `tenant_id` / `seller_id`.
- Аудит действий (минимум: кто/когда/что изменил по ключевым сущностям).

## E1 — Справочники и склад

- Склад, зоны, ячейки; статусы мест и товара.
- SKU: габариты, баркоды, связь с внешними ID WB (`nmID`, `vendorCode`, баркод и т.д. после импорта).

## E2 — Операции (внутренний контур)

- Заявки: приёмка, размещение, отгрузка (жизненный цикл статусов).
- Остатки: только через проведённые операции (инвариант BRD).
- Резерв под отгрузку (MVP): удержание количества по строке заявки с назначенной ячейкой в `draft`/`submitted` (`inventory_reservations`), «доступный» остаток для transfer и API балансов; удаление строки в `draft` — `DELETE .../lines/{id}` (снятие резерва); дальнейшее — см. `docs/NEXT_TASKS_RU.md` (кандидаты в issues).

## E3 — Биллинг v1

- Тариф: руб/литр/день + операционные услуги (приёмка, отгрузка и т.д. — уточнить таблицей тарифов).
- Начисления: связь с событием операции и снимком литр‑дней.

## E4 — Интеграция Wildberries (импорт)

- Хранение зашифрованных/защищённых токенов селлера (per seller).
- Клиент API: контент `POST /content/v2/get/cards/list`; поставки `POST /api/v1/supplies`, `GET .../supplies/{id}`, `GET .../goods`.
- Периодический sync + ручная кнопка «Обновить».
- Маппинг карточек и позиций поставок на внутренние SKU.

## E5 — Кабинет селлера (UI)

- Создание заявок (приём/отгрузка).
- Экраны приёмки и отгрузки (статусы, состав).
- Просмотр импортированных поставок WB (без создания в WB).

## E6 — Печать

- Шаблон этикетки 58×40; печать через браузер или ESC/POS (отдельная задача на исследование драйвера).

## Порядок внедрения (рекомендация)

1. E0 → E1 → E2 (скелет операций)  
2. Параллельно E4 (с моками)  
3. E5 поверх E2/E4  
4. E3 после стабильных событий операций  
5. E6 точечно по первому сценарию «нужна этикетка»
