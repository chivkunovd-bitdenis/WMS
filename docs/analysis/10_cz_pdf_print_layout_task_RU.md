# CZ print layout task — печать КМ как в PDF Честного знака

> **Назначение:** ТЗ на исправление печати ЧЗ после подтверждённого расхождения: в PDF ЧЗ этикетка выглядит как полноценный стикер, а из WMS печатается только DataMatrix и хвост кода.
> **Дата фиксации:** 2026-06-30.
> **Приоритет:** P0/P1 для ЧЗ-печати, потому что оператор получает физически неверную наклейку.

---

## Контекст

Пользователь показал два состояния одного сценария:

1. **PDF Честного знака:** этикетка 58×40 с DataMatrix слева и служебным/брендовым блоком справа.
2. **Фактическая печать из WMS:** на наклейке печатается только квадрат DataMatrix и короткий хвост КМ снизу.

Это не похоже на проблему принтера. В текущем коде WMS не печатает исходную PDF-этикетку и не воспроизводит её шаблон. WMS берёт CIS/КМ, заново генерирует DataMatrix и строит свою минимальную HTML-этикетку.

Ключевой текущий код:

- `frontend/src/utils/printMarkingCodeLabel.ts`
  - `TAPE_PAGE_CSS`
  - `renderDataMatrixDataUrl`
  - `buildCzLabelHtml`
  - `printMarkingCodeTape`
- `frontend/src/components/MarkingPrintDialog.tsx`
  - вызывает `printMarkingCodeTape(...)`
- `frontend/src/utils/markingPrintPresets.ts`
  - управляет лентой блоков `cz` / `label`

---

## Сейчас (баг)

`buildCzLabelHtml()` строит ЧЗ-блок так:

```html
<section class="label label--cz">
  <div class="matrix"><img ... /></div>
  <p class="tail">...</p>
</section>
```

CSS для `.label--cz` центрирует DataMatrix и хвост:

- страница `58mm 40mm`;
- DataMatrix `28mm × 28mm`;
- текст справа отсутствует;
- служебный блок из PDF отсутствует;
- оригинальная раскладка PDF не используется.

Итог: печать из WMS визуально и функционально не соответствует PDF-этикетке ЧЗ.

---

## Цель

Сделать печать блока `ЧЗ` в WMS визуально соответствующей PDF-этикетке Честного знака для термоформата 58×40:

- DataMatrix расположен как в PDF, не просто по центру пустой этикетки;
- справа присутствует служебный блок/текстовая часть, как в PDF-шаблоне;
- этикетка не обрезается и не съезжает на физической печати;
- текущий конструктор ленты `ШК ВБ / ЧЗ` сохраняется;
- блок `ШК ВБ` не ломается.

---

## Что Сделать

### 1. Исправить HTML/CSS шаблон ЧЗ-этикетки

В `frontend/src/utils/printMarkingCodeLabel.ts` заменить минимальный `buildCzLabelHtml()` на полноценный шаблон 58×40:

- общий контейнер остаётся `.label label--cz`;
- внутри нужен layout с двумя зонами:
  - левая зона: DataMatrix;
  - правая зона: служебный блок, визуально соответствующий PDF ЧЗ;
- хвост/маска КМ не должен быть единственным текстом;
- все размеры задавать в `mm`, не в пикселях;
- `overflow: hidden` допустим только как защита, но контент штатно должен помещаться без обрезки.

Не делать “примерно красивую карточку”. Нужно попасть в практический формат печати, похожий на PDF: компактная этикетка, DataMatrix + правый информационный блок.

### 2. Разобраться с источником PDF

Проверить, есть ли в текущем потоке импорта ЧЗ доступ к исходной PDF-разметке:

- если PDF загружается и можно сохранить/использовать страницу/фрагмент этикетки, описать и реализовать путь печати исходного PDF-фрагмента;
- если в системе после импорта остаётся только CIS/КМ, тогда воспроизвести PDF-шаблон на фронте по данным КМ.

Важно: нельзя оставлять текущий вариант “мы генерим только DataMatrix”, если пользователь ожидает PDF-этикетку.

### 3. Сохранить конструктор печати

Не ломать текущую модель:

- `MarkingPrintDialog` остаётся единой точкой печати;
- `layout.units` по-прежнему поддерживает произвольную ленту `cz` / `label`;
- `cz` теперь печатает полноценную ЧЗ-этикетку;
- `label` по-прежнему печатает WB/товарную термоэтикетку.

### 4. Проверить физический размер

Для ЧЗ-блока:

- `@page { size: 58mm 40mm; margin: 0; }` оставить или скорректировать только если нужен другой реальный формат;
- каждая этикетка должна занимать ровно одну страницу;
- не должно быть лишнего масштабирования браузером;
- не должно быть “усадки по содержимому”;
- DataMatrix должен оставаться читаемым сканером.

## Acceptance

1. Открываем печать ЧЗ для товара с КМ.
2. Выбираем блок `ЧЗ` в конструкторе.
3. Печатаем на 58×40.
4. На физической наклейке видна полноценная ЧЗ-этикетка:
   - DataMatrix;
   - правый служебный/текстовый блок как в PDF;
   - нет пустой этикетки с одним квадратом;
   - нет обрезания снизу/сбоку;
   - текст не уезжает за пределы наклейки.
5. Лента `ШК ВБ + ЧЗ` печатает две отдельные этикетки в выбранном порядке.
6. `ШК ВБ` печатается как раньше и не превращается в ЧЗ-шаблон.
7. Повторная печать и печать из упаковки используют тот же исправленный шаблон.

## Риски

- Если исходный PDF после импорта не хранится, невозможно “напечатать ровно тот PDF” без восстановления шаблона по данным.
- Если официальный шаблон ЧЗ отличается по типам товаров/маркетплейсам, нужен configurable template, а не один жёсткий layout.
- Если браузер/драйвер принтера масштабирует страницу, даже правильный CSS может печататься с отступами. Это надо проверить на реальном принтере 58×40.
