# Пакет GitHub Issues (после `gh auth login`)

На машине разработчика выполни: `gh auth login`, затем из корня репозитория:

```bash
gh issue create --title "[FEAT] E0: мультитенантность + пользователи + роли" --label "ready" --body-file - <<'EOF'
## User story
Как владелец фулфилмента, я хочу изолировать данные своих селлеров и управлять доступом, чтобы не было утечек между клиентами.

## Acceptance criteria
- Given новый фулфилмент (тенант), When он создаёт организацию и первого админа, Then данные недоступны другим тенантам.
- Given пользователь селлера, When он логинится, Then видит только данные своего селлера в рамках тенанта.

## E2E
- (после UI) Логин под двумя тенантами — нет пересечения списков.

## Примечание
Секреты только env; без хардкода URL/ключей.
EOF
```

Повтори по эпикам из `docs/BACKLOG_EPICS_RU.md` или создай Issues вручную в UI GitHub с тем же содержанием.

**Без авторизации `gh` агент не может создать issues с твоего Mac автоматически** — только подготовить текст (этот файл + epics).
