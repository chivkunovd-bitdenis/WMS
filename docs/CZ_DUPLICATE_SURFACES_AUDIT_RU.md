# Аудит дублей поверхностей ЧЗ (FINAL-02 / T-I1)

Дата: 2026-06-28. Источник постановки: `CHESTNY_ZNAK_UX_FIXES_RU.md` § T-I1.

Цель: на каждую функцию (лента, список кодов, импорт) — **одна каноническая поверхность**; остальное — превью + ссылка или удаление.

---

## 1. Лента расхода (две ленты)

| Поверхность | Файл | Роль |
|-------------|------|------|
| **Канон** | `HonestSignLedgerPage.tsx` | Полная лента: фильтры, пагинация, экспорт (LEDGER-*), все колонки |
| Превью | `HonestSignPoolPage.tsx` таб «Лента» | 5 последних событий + кнопка «Вся лента пула» |

**Статус:** ✅ устранено в **POOLCARD-03** — из карточки пула убрана полная таблица с фильтрами; осталось превью + ссылка на `…/honest-sign/ledger?pool_id=…`.

**FINAL-02:** превью переведено на `ledgerEventLabel` из `markingStatus.ts` (как на канонической странице).

**Не дубль:** drawer «История кода» на табе «Коды» — события **одного** КМ (`GET …/codes/{id}/history`), не замена ленты пула.

## 2. Список кодов (два списка)

| Поверхность | Файл | API | Роль |
|-------------|------|-----|------|
| **Канон** | `HonestSignPoolPage.tsx` таб «Коды» | `GET …/pools/{id}/codes` | Фильтр по статусу, поиск по хвосту КМ, экспорт CSV, история кода |
| Сирота | `MarkingProductCodesDialog.tsx` | `GET …/products/{id}/codes` | Диалог по товару + печать; **нигде не подключён** |

**Статус:** ✅ дубль снят — `MarkingProductCodesDialog` помечен deprecated и **не монтируется** в роутинге. Канон — таб «Коды» карточки пула. Файл диалога оставлен до отдельного тикета на удаление (SHARED-01 ссылался на него для словаря; backend-эндпоинт по товару может пригодиться для каталога).

**FINAL-02:** таб «Коды» — статусы через `codeStatusLabel` (единый словарь с лентой).

**Follow-up (не блокер):** если понадобится «коды по товару» из каталога — открывать карточку пула с `?tab=codes`, а не второй диалог.

## 3. Импорт кодов (два входа)

| Поверхность | Файл | Роль |
|-------------|------|------|
| **Канон** | `MarkingImportDialog.tsx` из `HonestSignScreen` | Рабочий импорт: превью, GTIN-группы, submit |
| Заглушка | маршрут `ff/honest-sign/import` → `HonestSignImportPage` | Статическая заглушка «T0.8 в работе» — **второй вход без функции** |

**Статус:** ✅ устранено в **FINAL-02** — маршрут `/import` делает `Navigate` на список пулов (`…/honest-sign`); импорт только через кнопку «Загрузить коды» / «Догрузить» → тот же диалог.

**Не дубль:** несколько кнопок «Загрузить» / «Догрузить» на экране списка — один и тот же `MarkingImportDialog`, разные точки входа (норма для CTA).

**Follow-up:** **CROSS-04** — «Догрузить» с контекстом пула (предзаполнение GTIN/title в импорте).
