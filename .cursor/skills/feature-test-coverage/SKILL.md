# Feature test coverage (WMS)

## Когда включать

Когда нужно быстро собрать блок `### Test coverage` для issue или PR (удобно любому агенту/человеку). **Жёсткая проверка** — в CI (`scripts/ci/check_pr_test_coverage.py`), не в порядке вызова Cursor-ролей.

## Источники (читать по порядку)

1. `docs/MVP_DECISIONS_RU.md` — границы MVP и запреты.
2. `docs/IMPLEMENTED_PRODUCT_SCENARIOS_EN.md` — смысл сценариев.
3. `docs/IMPLEMENTED_PRODUCT_SCENARIOS_TEST_CASES_EN.md` — канонические `TC-Sxx-yyy` и ожидаемое поведение.

При конфликте issue vs MVP — приоритет у MVP; отразить в `Notes` таблицы.

## Самопроверка перед отправкой (качество, не форма)

Пока не выполнено — **дополнить** таблицу, не сокращать «чтобы прошло»:

- [ ] У каждой строки с **Y** в **Notes** есть: действие пользователя → **видимый** результат при успехе (текст/экран/ошибка).
- [ ] Для затронутых ограничений по роли/статусу/данным — в **Notes** явно **негатив** или «должно быть заблокировано».
- [ ] Минимум **две** строки с разными `TC-...` (или одна Y и одна N с обоснованием в Notes), не одна строка-заглушка.
- [ ] В секции есть **несколько** формулировок уровня Given / When / Then (RU/EN) или явные **negative / restriction** — иначе CI отклонит как слишком короткий/пустой блок.

## Выход (артефакт)

Блок с заголовком **точно**:

```markdown
### Test coverage

| TC-ID | Title (short) | Applies (Y/N) | Notes |
|-------|-----------------|---------------|-------|
| TC-S06-001 | … | Y | |
| TC-NEW-001 | … | Y | Given … / When … / Then …; Negative: … |
```

- **Y** — кейс затрагивается задачей.
- Новое поведение без строки в EN-доке — **`TC-NEW-00n`** с Given/When/Then и негативами в `Notes`.

Скопируйте тот же блок в **описание PR** — CI читает только PR.

## После merge новых `TC-NEW-*`

При необходимости дополнить `docs/IMPLEMENTED_PRODUCT_SCENARIOS_TEST_CASES_EN.md` в том же PR, чтобы каталог не расходился с таблицей.

## Playwright

В заголовке `test()` или комментарии указывать **TC-ID** из таблицы (требование `AGENTS.md` и скрипт `check_e2e_tc_mentions.py` при изменении спек).
